# Installation Guide

## Prerequisites

- PHP 8.4+
- Composer
- Node.js & npm
- A GitHub App (not OAuth App)
- SQLite or MySQL/PostgreSQL

## Local Development Setup

### 1. Clone and Install Dependencies

```bash
git clone git@github.com:datashaman/pageant.git
cd pageant
composer install
npm install
```

### 2. Environment Configuration

```bash
cp .env.example .env
php artisan key:generate
```

### 3. GitHub App Configuration

Create a GitHub App at https://github.com/settings/apps with the following settings:

**Callback URL:**
```
https://your-domain.com/auth/github/callback
```

**Webhook URL:**
```
https://your-domain.com/webhooks/github
```

**Permissions required:**

| Permission | Access |
|------------|--------|
| Contents | Read & Write |
| Issues | Read & Write |
| Pull requests | Read & Write |
| Metadata | Read-only |
| Checks | Read-only |
| Commit statuses | Read-only |

**Events to subscribe to:**

- Issues
- Issue comment
- Pull request
- Pull request review
- Push

**Add these to your `.env`:**

```env
GITHUB_CLIENT_ID=your-github-app-client-id
GITHUB_CLIENT_SECRET=your-github-app-client-secret
GITHUB_APP_ID=your-github-app-id
GITHUB_APP_PRIVATE_KEY_PATH=storage/github-app-private-key.pem
GITHUB_WEBHOOK_SECRET=your-webhook-secret
```

Download the GitHub App's private key and save it to `storage/github-app-private-key.pem`.

### 4. Database

```bash
php artisan migrate
```

The default database is SQLite. To use MySQL or PostgreSQL, update `DB_CONNECTION` and related variables in `.env`.

### 5. OAuth (Passport) Setup

Pageant uses Laravel Passport for OAuth 2.1 authentication on the MCP server endpoint.

```bash
php artisan passport:keys
```

This generates `storage/oauth-private.key` and `storage/oauth-public.key`.

### 6. Build Frontend Assets

```bash
npm run build
```

### 7. Run the Application

```bash
composer run dev
```

Or individually:

```bash
php artisan serve
npm run dev
```

## Production Deployment

### Deploy Script (Laravel Forge)

```bash
$CREATE_RELEASE()

cd $FORGE_RELEASE_DIRECTORY

$FORGE_COMPOSER install --no-dev --no-interaction --prefer-dist --optimize-autoloader
$FORGE_PHP artisan optimize
$FORGE_PHP artisan storage:link
$FORGE_PHP artisan migrate --force

npm ci || npm install
npm run build

$ACTIVATE_RELEASE()

$RESTART_QUEUES()
```

### First Deploy — Additional Steps

After the first deploy, run once on the server:

```bash
php artisan passport:keys
```

This creates the OAuth encryption keys in the shared `storage` directory. They persist across zero-downtime releases.

### CSRF Exclusions

The MCP and OAuth routes are excluded from CSRF verification in `bootstrap/app.php`. This is required because MCP clients connect via stateless HTTP and don't send CSRF tokens.

## Connecting an MCP Client

### Claude Code

Pageant exposes three MCP servers:

```bash
# GitHub operations (issues, PRs, branches, labels, CI, search)
claude mcp add --transport sse pageant-github https://your-domain.com/mcp/github

# Pageant operations (repos, projects, work items, agents, skills, skill registry)
claude mcp add --transport sse pageant https://your-domain.com/mcp/pageant

# Worktree operations (file ops, shell commands, git) — used internally by agents
claude mcp add --transport sse pageant-worktree https://your-domain.com/mcp/worktree
```

Claude Code will handle the OAuth 2.1 flow automatically — it opens a browser for authorization, then uses the token for subsequent requests.

### Other MCP Clients

Both MCP servers support OAuth 2.1 discovery. Clients can discover endpoints via:

- `GET /.well-known/oauth-protected-resource` — Resource metadata
- `GET /.well-known/oauth-authorization-server` — Authorization server metadata (token endpoint, authorization endpoint, registration endpoint)

The server uses the `mcp:use` scope.

## Environment Variables

### Required

| Variable | Description |
|----------|-------------|
| `APP_KEY` | Application encryption key (generated by `php artisan key:generate`) |
| `APP_URL` | Full application URL |
| `GITHUB_CLIENT_ID` | GitHub App client ID |
| `GITHUB_CLIENT_SECRET` | GitHub App client secret |
| `GITHUB_APP_ID` | GitHub App ID |
| `GITHUB_APP_PRIVATE_KEY_PATH` | Path to GitHub App private key PEM file |
| `GITHUB_WEBHOOK_SECRET` | Secret for verifying GitHub webhook signatures |

### Optional

| Variable | Description | Default |
|----------|-------------|---------|
| `DB_CONNECTION` | Database driver | `sqlite` |
| `DB_DATABASE` | Database name/path | `database/database.sqlite` |
| `QUEUE_CONNECTION` | Queue driver | `database` |
| `ALLOWED_EMAILS` | Comma-separated email allowlist to restrict login | *(empty = allow all)* |
| `SMITHERY_API_KEY` | Smithery API key for skill registry search | — |
| `PASSPORT_PRIVATE_KEY` | Passport private key (alternative to key files) | — |
| `PASSPORT_PUBLIC_KEY` | Passport public key (alternative to key files) | — |
| `PASSPORT_CONNECTION` | Passport database connection | default |
